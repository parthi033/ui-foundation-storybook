# pn-button Component

The `pn-button` is a versatile web component from the UI Foundation library that provides a consistent button interface across your application.

## Overview

This component offers different button types to support various use cases in your UI:

- **Primary**: For main actions and calls-to-action
- **Secondary**: For supporting actions  
- **Disabled**: For unavailable actions

## API Reference

### Properties

| Property | Type   | Default     | Description                                    |
|----------|--------|-------------|------------------------------------------------|
| `type`   | string | `"primary"` | The button style (primary, secondary, disabled) |
| `label`  | string | `""`        | The text content displayed on the button      |

## Usage Examples

### Basic Usage

```html
<pn-button type="primary" label="Click me"></pn-button>
```

### Different Types

```html
<!-- Primary button for main actions -->
<pn-button type="primary" label="Save Changes"></pn-button>

<!-- Secondary button for supporting actions -->
<pn-button type="secondary" label="Cancel"></pn-button>

<!-- Disabled button for unavailable actions -->
<pn-button type="disabled" label="Processing..."></pn-button>
```

## Button Types

### Primary Button
Primary buttons are used for the most important actions in your interface, such as:
- Submit forms
- Confirm actions
- Primary navigation

### Secondary Button
Secondary buttons are used for less critical actions:
- Cancel operations
- Secondary navigation
- Alternative actions

### Disabled Button
Disabled buttons indicate that an action is temporarily unavailable:
- Form validation pending
- Processing states
- Insufficient permissions

## Best Practices

### Do's ✅

- Use primary buttons for the most important action on a page
- Use secondary buttons for supporting actions
- Keep button labels clear and concise
- Use disabled state to prevent invalid actions

### Don'ts ❌

- Don't use multiple primary buttons in the same context
- Don't use vague labels like "Click here" or "Submit"
- Don't rely solely on color to convey meaning
- Don't make buttons too small for touch targets

## Accessibility

The `pn-button` component includes built-in accessibility features:

- **Keyboard navigation**: Supports focus and activation via keyboard
- **Screen reader support**: Proper labeling and role attributes
- **Focus indicators**: Clear visual focus states
- **Touch targets**: Adequate size for touch interaction

## Integration

To use this component in your project:

1. **Install the UI Foundation library**:
   ```bash
   npm install @parthi033/ui-foundation
   ```

2. **Import the component**:
   ```javascript
   import '@parthi033/ui-foundation/dist/components/pn-button';
   ```

3. **Use in your HTML**:
   ```html
   <pn-button type="primary" label="Get Started"></pn-button>
   ```

## Browser Support

The `pn-button` component supports all modern browsers that support Custom Elements:

- Chrome 54+
- Firefox 63+
- Safari 10.1+
- Edge 79+
