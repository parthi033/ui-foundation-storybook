# pn-button Component

A versatile button component with icon support and multiple variations, built with Stencil as a framework-agnostic web component.

## Overview

The `pn-button` component offers comprehensive button functionality with:

- **Multiple button types**: primary, secondary, tertiary, white, and disabled
- **Icon support**: left, right, or icon-only configurations
- **Style variations**: rounded corners and shadow effects
- **Framework compatibility**: Works with React, Vue, Angular, and vanilla HTML
- **Icon library agnostic**: Compatible with Font Awesome, Heroicons, Lucide, Bootstrap Icons, etc.

## API Reference

### Properties

| Property    | Type    | Default     | Description                                          |
|-------------|---------|-------------|------------------------------------------------------|
| `label`     | string  | `""`        | Text displayed on the button                        |
| `type`      | string  | `"primary"` | Button style variant (primary, secondary, tertiary, white, disabled) |
| `rounded`   | boolean | `false`     | Apply rounded corners                                |
| `shadow`    | boolean | `false`     | Add shadow effect                                    |
| `iconLeft`  | string  | `""`        | Icon class for left-side icon (e.g., "fas fa-save") |
| `iconRight` | string  | `""`        | Icon class for right-side icon                      |
| `iconOnly`  | boolean | `false`     | Display only icon without text                       |

## Usage Examples

### Basic Usage

```html
<pn-button label="Click me" type="primary"></pn-button>
```

### Button Types

```html
<!-- Primary for main actions -->
<pn-button label="Save Changes" type="primary"></pn-button>

<!-- Secondary for supporting actions -->
<pn-button label="Cancel" type="secondary"></pn-button>

<!-- Tertiary for subtle actions -->
<pn-button label="Learn More" type="tertiary"></pn-button>

<!-- White for colored backgrounds -->
<pn-button label="Get Started" type="white"></pn-button>

<!-- Disabled for unavailable actions -->
<pn-button label="Processing..." type="disabled"></pn-button>
```

### With Icons

```html
<!-- Icon on the left -->
<pn-button label="Save Document" type="primary" icon-left="fas fa-save"></pn-button>

<!-- Icon on the right -->
<pn-button label="Continue" type="secondary" icon-right="fas fa-arrow-right"></pn-button>

<!-- Icon only -->
<pn-button type="primary" icon-left="fas fa-heart" icon-only="true"></pn-button>
```

### Style Variations

```html
<!-- Rounded corners -->
<pn-button label="Rounded" type="primary" rounded="true"></pn-button>

<!-- With shadow -->
<pn-button label="Shadow" type="primary" shadow="true"></pn-button>

<!-- Combined features -->
<pn-button 
  label="Enhanced Button" 
  type="primary" 
  icon-left="fas fa-star"
  rounded="true" 
  shadow="true">
</pn-button>
```

## Best Practices

### Do's ✅

- Use primary buttons for the most important action on a page
- Use secondary buttons for supporting actions  
- Use tertiary buttons for subtle, less prominent actions
- Keep button labels clear and action-oriented
- Use icon-only buttons sparingly and with clear context
- Combine rounded and shadow effects thoughtfully

### Don'ts ❌

- Don't use multiple primary buttons in the same context
- Don't use vague labels like "Click here" or "Submit"
- Don't rely solely on color to convey meaning
- Don't make touch targets too small (minimum 44px)
- Don't overuse shadow effects

## Accessibility

The `pn-button` component includes built-in accessibility features:

- **Keyboard navigation**: Full keyboard support with focus indicators
- **Screen reader support**: Proper ARIA attributes and semantic HTML
- **Touch targets**: Adequate size for touch interaction (minimum 44px)
- **High contrast**: Color combinations meet WCAG guidelines
- **Focus management**: Clear visual focus indicators
